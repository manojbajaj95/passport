What kind of demo are we building?

This shapes everything — API design, tooling, how you'd present it to others

A

REST API + CLI client

A server exposing HTTP endpoints. A small CLI tool (e.g. passport init, passport register) that an agent or developer runs. Fast to build, easy to demo in a terminal.

B

REST API + Web Dashboard

A server with HTTP endpoints, plus a browser UI showing registered agents, ownership status, passport details, and claiming flows. More visual, better for showing to stakeholders.

C

API only (no client)

Just the server — well-documented REST endpoints. Agents call it directly. Simple, focused on the protocol. Easiest to keep clean and extend later.
